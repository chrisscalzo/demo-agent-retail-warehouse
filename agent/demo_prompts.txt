SECTION 4: CROSS-STATE COMPARISON PROMPTS (5)
Prompts that invite the agent to compare policies across CA/NV/AZ
────────────────────────────────────────────────────────────────

1.  How does the overtime policy differ between our California, Nevada, and Arizona warehouses?
    (Looking for: CA has daily OT after 8 hrs + double time after 12; NV weekly only [unless below 1.5x min wage]; AZ weekly only — no daily OT)

2.  If a worker gets injured at work, how does the deadline for reporting to the employer's insurance company compare across CA, NV, and AZ?
    (Looking for: CA = 1 day to provide DWC-1; NV = 7 days employee reporting; AZ = 10 days employee reporting)

3.  Do workers get to choose their own doctor for a work injury in all three states, or does the company control that?
    (Looking for: CA = employee chooses from MPN after initial; NV = employer panel but employee can switch; AZ = employer directs first 90 days)

4.  Which of our states has the strictest heat illness requirements, and what's different about each one?
    (Looking for: CA = most prescriptive — §3395 mandatory written plan, shade at 80°F; NV = guidance-based, 90°F protocols; AZ = federal OSHA General Duty + extreme heat/monsoon overlays)

5.  How do the OSHA enforcement structures differ between California, Nevada, and Arizona — and what does that mean for our safety programs?
    (Looking for: CA = Cal/OSHA state plan, DOSH enforcement, requires IIPP; NV = Nevada OSHA state plan, DIR, requires APP; AZ = Federal OSHA direct, AIC handles WC only — no state safety agency for private employers)

================================================================
USAGE NOTES FOR DEMO
================================================================

SECTION 1 (General): Use these to show the agent answering
from the general SharePoint knowledge base — no state filtering.
Good for: new hire demos, supervisors, HR questions.

SECTION 2 (State Generic): Use these to show the agent
personalizing answers based on the user's profile state.
Demonstrate the system prompt instruction:
  "When answering safety or labor questions, use documents
   from the user's state folder ({UserState})."
Good for: showing personalization without the user having
to specify their state every time.

SECTION 3 (State Explicit): Use these to show the agent
overriding the user profile when a state is named.
Demonstrate the instruction:
  "If the user mentions a specific state by name, use
   documents from that state's folder instead of the
   user's profile state."
Good for: managers overseeing multiple facilities,
supervisors asking about other locations.

SECTION 4 (Cross-State Comparison): Use these to show
the agent synthesizing across multiple state documents.
These are high-impact demo moments — they show the agent
doing real knowledge work, not just document lookup.
Expected answers are noted in parentheses for demo prep.

SECTION 5 (Supervisor List Queries): Use these to show
the agent reading from the SharePoint "Safety Issue Log"
list as a supervisor. Demonstrate cross-location visibility,
severity filtering, near-miss identification, and trend
analysis. Best shown logged in as a Manager/Supervisor
persona. Connect list via Power Automate or SharePoint
connector in Copilot Studio.

SECTION 6 (Team Member List Queries): Use these to show
the agent reading from the list from a team member's
perspective — status checks on specific issues, shift
awareness, and location-specific queries. Same connector
as Section 5; the agent persona/topic determines the
answer framing.

SECTION 7 (New Submissions — Slot-Filling): Use these to
show the "Log Safety Issue" topic in action. The agent
should ask for:
  • Reporter name
  • Warehouse location (drop-down of 6 choices)
  • Issue type (Equipment / Environmental / Process /
    Near-Miss / Other)
  • Description of what you observed
  • Severity (Low / Medium / High / Critical)
Then confirm and submit via Power Automate flow.
These prompts are intentionally incomplete — they give
the agent a starting description so the slot-filling
conversation looks natural and realistic in the demo.
